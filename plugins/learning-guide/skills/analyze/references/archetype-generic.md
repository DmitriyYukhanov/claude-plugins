# Archetype: generic

Use when the input does not match any of the three specific archetypes (codebase, planning-session, refactor-plan).

## Section outline (defer to input shape)

For arbitrary documents:

1. **Introduction** — what this is and how to read it.
2. **Body sections** — mirror the input's heading hierarchy.
3. **Glossary** — terms encountered.
4. **Self-check** — 3–5 quiz items covering the most consequential claims.

## What to look for

- Heading hierarchy and table of contents — drive section structure directly.
- Cross-references between sections — propose `cross_ref_patterns`.
- Domain terms — populate the `glossary` array.

## Synthesis

If the input is sparse and no user-authored doc exists, plan to synthesize `tour-companion.md`.

## Tone

Neutral; mirror the input's voice.
